Maven是一个项目管理软件

1. Maven的依赖管理

依赖管理就是maven工程对jar包的管理

依赖范围：

范围 对于编译有效 对于测试有效 对于运行时有效 例子
compile √ √ √ spring-core
test - √ - junit
provided √ √ - servlet-api
runtime - √ √ JDBC驱动
system √ √ - 本地的

2. 项目的一键构建

指的是整个构建过程，使用 maven 一个命令可以轻松完成整个工作。

3. 仓库的种类和彼此关系

3.1 本地仓库

用来存储从远程仓库或中央仓库下载的插件和jar包，项目使用一些插件或 jar包，优先从本地仓库查找。

3.2 远程仓库

也叫私服，如果需要插件或者jar包，如果远程仓库没有，可以从本地上传或者中央仓库下载。

3.3 中央仓库

在 maven 软件中内置一个远程仓库地址 http://repo1.maven.org/maven2 ，含概大部分流行的开源项目构件。

4. Maven标准目录结构

5. Maven常用命令

  1. 编译

    mvn compile

  2. 测试

    mvn test

  3. 打包

    mvn package

  4. 安装

    mvn install

  5. 发布

    mvn deploy

    以上是默认的生命周期，测试会包含编译，打包包含编译和测试，以此类推。

  6. 清理

    mvn clean

清理独立为一个生命周期，清理掉之前的编译信息

  1. 运行

    tomcat:run
